## filters.txt format

Each non-blank, non-comment line in `filters.txt` must follow this syntax:

```text
[group][category] "regular_expression"
```

- **`[group]`** — sets a shared display color for every announcement that matches any expression in this group. The color applies across all windows.
- **`[category]`** — controls per-window show/hide visibility. An empty category tag `[]` maps internally to the name `Other/All`.
- **`"regular_expression"`** — a Python `re`-module pattern. See the [Python re documentation](https://docs.python.org/2/library/re.html) for syntax.

### Parsing rules

- Lines beginning with `#` are treated as comments and ignored.
- Blank lines are ignored.
- Multiple lines sharing the same `[group][category]` tag are OR'd together — an announcement matches the category if it satisfies **any** of those expressions.
- An `UNKNOWN` group with a `unmatchedString` category is automatically appended after the file is loaded. It catches every announcement that did not match any earlier expression. Its default color is `#FF0`.

### Matching order

Groups are checked in the order of their **first appearance** in the file. Once the correct group is identified, its categories are checked sequentially from top to bottom. The first matching expression wins; subsequent expressions in the same file are not evaluated.

```text
# Matching stops at the first successful group+category hit.
# Placing a catch-all at the top of the file would absorb everything.
[everything][] "(.+)"
```

### Internal parsing regex

The application uses the following regular expression (from `Filters.py`) to parse each filter line:

```python
\[(?P<group>\w+)\]\[(?P<category>\w+|\s*)\]\s*\"(?P<expression>.+)\"
```

Named capture groups:
- `group` — the group tag value
- `category` — the category tag value (may be empty/whitespace for `Other/All`)
- `expression` — the raw regular expression string

### Example lines

```text
# Comment — this line is ignored

[battle_minor][hitevents_miss] "[tT]he (.+) misses [tT]he (.+)!"
[battle_minor][hitevents_miss] "[tT]he flying (.+) misses (.+)!"

# Empty category — maps to "Other/All"
[fishing][] "[tT]here is nothing to catch in the (.+)\."

# Separate group for a specific intruder type
[intruders_Thief][intruders] "Thief!  Protect the hoard from skulking filth!"
```

## wordcolor.txt format

Each non-blank, non-comment line in `wordcolor.txt` must follow this syntax:

```text
[group][colortag] "word1,word2,word3"
```

- **`[group]`** — must match a group name from `filters.txt`. The special group `General` applies its word list to **all** announcement groups.
- **`[colortag]`** — references a color name defined in the `[Colors]` section of `Settings.cfg`.
- **`"word1,word2,word3"`** — a comma-separated list of words to highlight with that color. Matching is case-sensitive.

### Lookup behavior

When an announcement is rendered, the app first checks for a matching word in the announcement's specific group, then falls back to the `General` group. Group-specific entries supplement (rather than replace) the `General` word list.

### Internal parsing regex

The application uses the following regular expression (from `WordColor.py`) to parse each word-color line:

```python
\[(?P<group>\w+)\]\[(?P<colorName>\w+|\s*)\]\s*\"(?P<word_list>.+)\"
```

Named capture groups:
- `group` — the group tag value
- `colorName` — the color tag name (must match a key in `[Colors]`)
- `word_list` — comma-separated word list

### Example lines

```text
# General words — highlighted in all groups
[General][red] "cancels,mandate"

# Same word with different colors per group
[battle][green] "Urist McColored"
[masterpiece][olive] "Urist McColored"

# Dwarf profession highlighting (General group)
[General][silver] "Miner"
[General][yellow] "Woodworker,Carpenter,Bowyer,Woodcutter"
```
